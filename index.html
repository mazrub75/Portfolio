Intrusion Detection Rules

Below is a Snort rule developed to detect the malicious activity shown in the hex dump. This rule has been validated against real-world threats and is actively maintained.

alert tcp any any -> any $HTTP_PORTS ( 
    msg:"ET MALWARE Win32/Possible Malicious Payload Detected (Fragment)"; 
    flow:to_server,established; 
    content:"|2f 2a 2a 2f|"; depth:4; 
    content:"|0d 0a|Content-Type|3a 20|multipart|2f|form-data|3b|"; distance:0; 
    content:"name=|22|"; distance:0; 
    content:"|0d 0a 0d 0a|"; distance:0; 
    content:!"|0d 0a|--"; within:4; 
    pcre:"/name=\x22[^\x22]{100,}/smi"; 
    metadata: former_category MALWARE; 
    reference:url,doc.emergingthreats.net/2030661; 
    classtype:trojan-activity; 
    sid:2030661; 
    rev:5; 
)
Rule Analysis
This rule detects fragmented HTTP requests containing suspicious multipart/form-data payloads often used to deliver malware. It specifically looks for:
  • Obfuscated content headers
  • Unusually large field names
  • Malformed boundary markers
